# Café Simulator — Accessibility-First Design Addendum
## Accessibility Philosophy
This game is designed as:
- A fully text-based simulation experience optimized for screen reader use (NVDA on Windows).
- No reliance on visuals, animations, or visual timing cues
- Every game state is described in structured, readable text
- Actions are always explicitly announced
- Feedback is immediate and descriptive
- Consistent menu layouts for navigation predictability
## How feedback will work (important)
The game will use structured system messages like:
- “Time advanced to Afternoon.”
- “Today's weather prediction is: Sultry. Customer traffic is reduced.”
- “New order received: Coffee x2.”
- “Order completed successfully. You recieved coins.”
This ensures NVDA reads everything cleanly and consistently.
## LEVEL SYSTEM (Expanded and Refined)
### Level 1 — Park (Tutorial Level)
Environment:
- Open park stall
- Casual passersby
- Relaxed but inconsistent traffic
Gameplay focus:
- Basic order handling
- Learning customer types
- Understanding time + weather
Systems active:
- NPC spawning (low rate)
- Basic weather effects
- Simple menu (Coffee, Cookies)
Customer behavior:
- Walkers (coffee)
- Kids (cookies, milk)
- Families (light mixed orders)
Key learning:
“How do I respond to demand when it appears randomly?”

### Level 2 — Leisure Zone
- Slightly higher NPC variety
- More snack-based demand
- Mild crowd bursts during evening
Possible Events:
- Small “event spikes” (random gatherings)
- Slightly more complex orders
Key learning:
“Demand is not constant — it comes in waves.”

### Level 3 — Market Street
- Fast turnover customers
- High order frequency
- Short decision time pressure
Key challenge:
Speed and accuracy matter more than exploration.

### Level 4 — Town Center
- Dense urban hub
- High variability
- Rush periods
- Mixed customer types constantly
Key challenge:
“Can you keep up when everything happens at once?”

## Phase 1 System Summary
- No persistent reputation
- No long-term customer memory. Temporary impressions only
- Focus: learning + adaptation

### Level 5 — Residential Neighbourhood
- Houses, repeat foot traffic routes
- Repeat customers begin appearing
- Families return
- Kids recognize previous orders
- Morning/evening routines matter
Key Challenge
early reputation memory (light persistence)

### Level 6 — Office Cafeteria
- predictable lunch rush
- time-sensitive demand spikes
Key challenge:
“If you miss timing, you lose business.”

### Level 7 — Mall outlet
- larger orders
- combo-based purchasing
- higher income per customer
Key learning:
reputation affects customer volume

### Level 8 — Banquet Hall
- bulk orders
- scheduled rushes
- high-pressure coordination
Key learning:
“One event can define your entire day.”

## Phase 2 System Summary
- Strong reputation system
- Repeat customers exist
- Predictable demand cycles
- Strategic optimization gameplay
